Pinning

Show pin buttons on tabs and groups. Pinned groups have a frozen name and a fixed set of member tabs; reclustering won't move pinned tabs out.

Show the bookmark ★ button on each group (bookmarks all tabs in a group to the Bookmarks Toolbar).

Clusters by content across the whole tab strip, ignoring current position. Identical or near-identical pages cluster together, and non-groupable tabs (about:newtab, about:debugging, etc.) sit at the end of whichever cluster they best match. When off, the original position-preserving algorithm is used.

When "Re-organize tabs" runs, also reorder the groups themselves by content similarity (greedy nearest-neighbor on group centroids), so related groups end up adjacent in the tab strip. When off, groups stay in their current left-to-right order.

Re-apply groups (rearranging the tab strip and creating Firefox tab groups) automatically every time the cluster recomputes.

When unchecked, group names don't auto-update and newly opened tabs appear below a separator until you click "Apply groups".

Hide Firefox's native tab strip

Firefox extensions can't hide the top tab strip directly. To rely only on the arctictab sidebar, edit your Firefox profile's userChrome.css:

  1. Open about:config and set toolkit.legacyUserProfileCustomizations.stylesheets to true.
  2. Open about:profiles and click Open Folder next to your current profile's Root Directory.
  3. Inside that folder, create a subfolder named chrome if it doesn't exist.
  4. Inside chrome/, create or edit userChrome.css and add: 
    #TabsToolbar { visibility: collapse !important; }
  5. Restart Firefox.

To restore the tab strip, remove that rule (or set the about:config pref back to false) and restart.
